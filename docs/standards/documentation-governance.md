# Documentation Governance Standard

How docs are organized and kept link-safe in this repo. Enforced (best-effort) by
`scripts/audits/verify-docs-integrity.mjs` (`docs:check`).

---

## 1. Temporal buckets (`YYYY-MM`)

Anything dated — incidents, RCA plans, audit findings, plans, retros — lives in a month bucket so
the tree stays navigable and old material self-archives:

```
docs/errors/2026-06/                      incident notes for June 2026
docs/errors/INDEX.md                      generated index of all incidents
docs/errors/rca/rca-plan-done/2026-06/    executed RCA plans
docs/plans/2026-06/                       active plans
docs/plans/done/2026-06/                  completed plans
docs/plans/worktrees/                     per-worktree plan state
docs/audits/findings/2026-06/             audit findings
docs/audits/INDEX.md                      generated audit index
docs/ai/retro/2026-06/2026-06.md          monthly retrospective
docs/meta/dreaming/                       meta-scan reports (cadence)
```

When a new month begins, create the new `YYYY-MM/` sibling — never reuse the previous month's bucket
for new dated material.

Non-dated standing docs (entry docs, standards, strategy, governance, the maturity rubric) live at
stable paths and are *not* bucketed.

---

## 2. Link-safety: backtick code-paths, never relative markdown links

> **Rule:** reference another file as an inline-code path — `` `docs/errors/INDEX.md` `` — **not** a
> relative markdown link like `[index](../errors/INDEX.md)`.

Why: a relative link breaks the moment a file moves to a different depth (e.g. into a deeper temporal
bucket), and `docs:check` then fails on a dangling link. A backtick code-path is stable text — the
reader resolves it from the repo root. This entire docs tree follows the rule, and the integrity
checker enforces it: any `[label](relative/path.md)` style internal link is a finding, and any
backtick path that points at a missing file is a finding.

External `https://` links and anchor-only links (`#section`) are allowed.

---

## 3. Indexes are generated, not hand-edited

`docs/errors/INDEX.md` and `docs/audits/INDEX.md` are regenerated by their verifiers
(`verify-learning-loop.mjs --write-index`, `verify-audit-loop.mjs --write-index`). Do not hand-edit
them; edit the source notes and regenerate.

---

## 4. The doc index

`docs/DOC_TOC.md` is the human table of contents. Add a row when you add a standing doc; keep it as
backtick code-paths.
